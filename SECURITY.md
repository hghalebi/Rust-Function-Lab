# Security Policy

## Supported versions

This repository is an educational project. Critical production guarantees are not
applicable, but we still care about responsible handling of issues and dependencies.

## Reporting a security concern

If you discover a security issue:

1. Open a private advisory or security issue in the repository.
2. Include:
   - steps to reproduce,
   - impact,
   - and environment details.

We will triage reports within a reasonable timeframe and coordinate fixes.

## Dependency updates

- Run `cargo update` regularly.
- Re-check with `scripts/quality.sh` after dependency changes.
